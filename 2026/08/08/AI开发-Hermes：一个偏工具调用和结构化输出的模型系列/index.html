AI开发-Hermes：一个偏工具调用和结构化输出的模型系列

发表于2026-08-08|更新于2026-08-08|AI开发
|字数总计:2.0k|阅读时长:8分钟|阅读量:

前言

Hermes 是 Nous Research 推出的模型系列。这里以 Hermes 3 为例，它不是单纯追求聊天口感，而是比较强调 agentic capabilities、function calling、structured output、多轮对话和代码能力。对于 AI 应用开发来说，这类能力往往比“闲聊是否漂亮”更关键。

比如做一个工具型助手，模型需要稳定输出 JSON、按函数签名组织参数、理解 system prompt 里的角色约束。如果这些能力不稳，后端就会花很多精力做兜底解析。

Hermes 3 的定位

Hugging Face 模型卡里提到，Hermes 3 是 Nous Research Hermes 系列的最新版本，相比 Hermes 2，在 Agent 能力、角色扮演、推理、多轮对话、长上下文一致性等方面都有改进。Hermes 3 也继续强化了函数调用和结构化输出能力。

这说明它更适合被当作应用组件，而不只是一个聊天 Demo。你可以把它放在需要工具调用、JSON 输出、复杂系统提示词的场景里试一试。

Prompt 格式要尊重模型习惯

Hermes 3 使用 ChatML 风格的 prompt format。很多模型都有自己的模板，如果调用时忽略模板，效果会明显下降。使用 transformers 时，应尽量通过 tokenizer 的 chat template 组织 messages，而不是手拼字符串。

函数调用也一样。模型卡给出的示例是把工具签名放到 system role，并要求模型用特定结构返回工具调用。开发时要让解析器和 prompt 格式保持一致。

本地部署怎么选

70B 级别模型对显存要求比较高，不适合普通笔记本直接跑全精度。模型卡给了 vLLM、SGLang、Docker Model Runner 等使用方式，也提供量化版本入口。

如果只是学习函数调用流程，可以先用小尺寸或量化模型验证 prompt、解析器和工具执行链路。等链路稳定，再换更大的模型做效果评估。

小结

Hermes 这类模型提醒我们：AI 应用不只看模型会不会聊天，还要看它能不能稳定遵守格式、调用工具、处理多轮上下文。做 Agent 或自动化工具时，结构化输出和函数调用经常是第一优先级。

参考资料：NousResearch/Hermes-3-Llama-3.1-70B 模型卡。
